# ActivitySummary

用于检索和整理粤港澳线下硬核技术活动的提示词与技能仓库。

这个仓库里有两样东西：

1. 一份可以直接拿去跑自动化的完整提示词
2. 一份给 Claude Code、Codex 这类代理调用的 skill

这个仓库更适合下面这种用法：

1. 先把真实活动捞出来
2. 再保留可核验、技术主题明确、公开可查的活动
3. 最后按固定模板输出主清单、候补链接和交通建议

## 适合什么场景

- 每天或每周跑一次活动汇总
- 想优先看深圳，也接受香港、广州、珠海、澳门的高含金量活动
- 想保留英文活动标题，但后面补一句中文解释
- 主清单为空时，仍然想看到深圳本地的接近候补
- 需要一份稳定、长期可复用的输出格式

## 快速改法

平时最常改的是提示词开头这几行：

- `地区范围`
- `时间范围`
- `固定起点`
- `重点活动平台`

平时先改这几行，后面大段正文一般保持原样。

提示词位置：

- `prompts/activity-summary-prompt.md`

skill 主文件位置：

- `SKILL.md`

## 这个仓库怎么分

- `prompts/activity-summary-prompt.md`
  - 直接给自动化或手动检索用的完整版提示词
- `SKILL.md`
  - skill 主文件，写的是触发条件、检索流程、筛选规则、输出规则
- `agents/openai.yaml`
  - skill 的展示元数据
- `references/output-template.md`
  - 固定输出模板
- `references/source-checklist.md`
  - 信息源优先级和搜索顺序
- `references/filtering-rubric.md`
  - 主清单、候补区、直接剔除的判断规则

## 现在这版的运行方式

现在这版已经补了几条约束：

1. 先查高信号来源，尤其是 Luma、Meetup、高校官网、学术组织
2. 找到足够活动就进入整理阶段
3. 来源池按候选池来用，搜索顺序更清楚
4. 主清单和候补区分开，结果结构更稳定

如果你在 Claude Code 里跑，这几条很重要。
