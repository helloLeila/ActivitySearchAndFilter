# Filtering Rubric

这个文件用来判断候选活动应该进主清单、候补区还是直接剔除。

## A. 进入主清单

必须同时满足：

1. 公开真实
2. 时间地点明确
3. 技术主题明确
4. 面向开发者、科研人群、学生或公众可见
5. 技术硬度达标

优先主清单的典型例子：

1. 高校 AI / CSE seminar
2. CCF / IEEE 纯技术研讨
3. 有明确技术议题的开源 meetup
4. 研究型招生开放日、RA/硕博宣讲

## B. 进入候补区

可以是真活动，但不满足主清单的“纯度”或“硬度”要求。

常见候补原因：

1. 主题接近，但不够纯计算机
2. 偏产业、偏产品，但仍有一定技术含量
3. 高校活动真实，但与计算机主题有一点距离
4. 时间地点明确，但技术深度略弱

候补区只给短链接，不展开长解释。

## C. 直接剔除

出现下面任一情况，直接不输出：

1. 商业宣讲
2. 招商、路演、老板圆桌
3. 亲子、徒步、市集、演出
4. 私密闭门，不公开报名
5. 时间或地点不明确
6. 纯概念空谈
7. 完全不属于技术主题

## D. 边缘判断

如果活动在“学术真实”与“主题不够纯”之间摇摆：

1. 不要硬塞进主清单
2. 放候补链接
3. 用最短原因词标明，比如：
   - `偏材料`
   - `偏生命科学`
   - `偏产品`
   - `偏产业论坛`
   - `商业味偏重`

## E. 含金量打分

### 高

1. 高校或学术组织主办
2. 议题具体
3. 技术对象清晰
4. 对开发者或科研人群有直接收益

### 中

1. 真实技术活动
2. 主题相关
3. 但技术深度略弱或混入少量非核心内容

### 低

1. 泛科技
2. 泛创新
3. 泛 AI 赋能
4. 内容无法判断具体技术价值

低含金量不要进主清单。
