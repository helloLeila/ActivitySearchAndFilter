# Source Checklist

先搜这些最容易出真活动的源头。

如果 skill 顶部配置块里写了“重点活动平台”，默认优先按那一行执行。当前默认重点平台是：`Luma、Meetup`。

这个清单是“优先候选池”，不是“必须逐个穷举完的执行清单”。
如果第一优先级已经找到足够活动，就可以停止，不需要把第二、第三优先级全部翻完。

## 第一优先级

1. Luma 官方活动页
2. Meetup 官方活动页
3. 高校官网、学院官网、研究院官网、官方 event calendar
4. CCF、ACM、IEEE 官网或官方公众号
5. 主办方官网

## 第二优先级

1. 腾讯、阿里、字节、华为技术社区
2. AWS 社区、OceanBase、Deepin、CANN
3. CNCF、K8s、Go、Rust、WebAssembly
4. LangChain、LLM 社区、PyTorch、TensorFlow、向量数据库、机器人社区

## 第三优先级

1. InfoQ、极客时间、SegmentFault、掘金、OSC、CSDN 线下活动
2. 前端早早聊、freeCodeCamp、OpenCloudOS、GitCode、开源之夏
3. 大湾区 AI、前端、Agent、具身智能社区

## 搜索顺序建议

1. 先扫 Luma 和 Meetup 上当前地区范围内的活动
2. 再搜核心城市的高校和学术组织
3. 再搜地区范围内其他城市的高校和高含金量活动
4. 再补其余城市与技术社区
5. 最后用开发者社区补小型 meetup

## 停机建议

满足任一条件即可停止扩展搜索：

1. 已确认 `3 场以上` 主清单活动
2. 已确认 `2 场以上高含金量活动` 且候补区已有内容
3. 连续几轮搜索没有新增真实候选

## 核验优先级

同一活动多平台出现时，优先保留：

1. 高校官网 / 学院官网 / 官方日历
2. 官方公众号原文
3. 主办方官网
4. Luma / Meetup
5. 媒体转发页

媒体稿、二手转发页只能辅助验证，不能作为唯一主链接。
