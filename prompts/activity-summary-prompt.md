# ActivitySummary Prompt

## 可配置头部

- 地区范围：深圳、广州、珠海、香港、澳门
- 时间范围：两周内（从本周一00:00至下周日23:59）
- 固定起点：深大地铁站A3/A4出口
- 重点活动平台：Luma、Meetup

使用规则：

1. 改地区时，优先只改 `地区范围` 这一行。
2. 改时间窗时，优先只改 `时间范围` 这一行。
3. 改重点平台时，优先只改 `重点活动平台` 这一行。
4. 如果顶部配置与正文中默认示例城市冲突，以顶部配置为准。
5. 正文里出现的“深圳、广州、珠海、香港、澳门”，默认视为当前配置版本的示例写法；执行时应按顶部配置解释，不需要逐段手改。

---

每天检索并汇总两周内（从本周一00:00至下周日23:59）深圳、广州、珠海、香港、澳门的线下真实硬核技术活动，目标是“更容易搜到真实可去的活动”，同时严格禁止编造、虚构、凑数。核心原则：先找到真实活动，再补充细节；宁可少量字段缺失并明确说明，也不要因为次要字段不全而把真实活动全部过滤掉。

一、收录底线（必须同时满足）
1. 必须是线下真实活动。
2. 必须能在公开网页上核验到，且至少有：活动名称、明确时间、明确城市/地点、主题说明。
3. 必须与计算机/AI/开发者/科研相关，且内容有技术密度，不是泛泛而谈。
4. 必须面向公众、学生、开发者或科研人群可见；纯私密闭门、仅邀请制且无公开说明的不收录。

二、硬核优先级
优先收录以下活动：
1. 高校计算机/AI学院、实验室、研究院的学术讲座、学者报告、Workshop、可旁听研讨会。
2. CCF、ACM、IEEE、顶级开源社区、技术委员会主办的纯技术活动。
3. Luma、Meetup、GitHub、开源社区上的纯技术开发者 Meetup。
4. 香港八大+岭南大学的研究硕博、RA、开放日、教授面对面咨询，但仅限研究型、学术型、院系官方活动。
5. 计算机发展史、AI算力、数字技术、机器人/具身智能相关的专业技术展览。

三、硬核判断标准
满足任一即可优先收录：
1. 有代码、Demo、命令行、实验、系统实现、工程落地、算法推导、模型训练/部署、HCI实验方法等具体内容。
2. 主题聚焦前端、Vue3、后端、全栈、软件工程、HCI、人机交互、数据可视化、人工智能、大模型、Agent、具身智能、机器人、AGI、计算机科研。
3. 主办方或讲者具有明确学术/技术权威性，如高校计算机/AI学院、CCF/ACM/IEEE、知名教授或资深工程团队。

四、永久排除
1. 厂商产品宣讲、解决方案推销、商业峰会、招商论坛、老板圆桌、政企高管论坛。
2. 文艺、市集、徒步、亲子科普、泛创业、培训招生、训练营、集训营、孵化器、圈层收费局。
3. 纯概念空谈、跨行业拼盘、无明确技术主题、无公开时间地点的活动。
4. 非计算机相关的数理化研讨会，除非与AI、机器人、计算成像、计算材料、HCI等计算主题直接相关。

五、地区与含金量策略
1. 深圳：可收录高含金量和中含金量活动。
2. 广州、珠海、香港、澳门：只收录高含金量活动。
3. 如果活动很真实、很硬核，但个别辅助信息缺失，不要直接剔除；优先保留，并明确写“官方未公开”。

六、信息源检索策略
必须优先检索这些来源，但不要求在最终结果里逐个平台机械证明“全都搜过”。将它们视为候选池：Luma、Meetup、InfoQ、极客时间、SegmentFault、掘金、开源中国OSC、CSDN线下活动、腾讯/阿里/字节/华为技术社区、AWS社区、OceanBase、Deepin、CANN、LangChain、LLM社区、PyTorch、TensorFlow、向量数据库、机器人社区、CNCF/K8s/Go/Rust/WebAssembly、CCF/ACM/IEEE、南科大/深大/港中深/华工/暨大/澳大官网与公众号、香港八大与岭南大学招生官网、前端早早聊、freeCodeCamp、OpenCloudOS、GitCode、开源之夏，以及大湾区AI/前端/Agent/具身智能社区。
优先使用最权威的一手来源：高校官网、学院官网、官方日历、官方公众号原文、Luma活动页、Meetup活动页、主办方官网。若同一活动多平台出现，只保留最权威来源。

七、输出格式（强制统一，优先可读性）
每次输出必须使用下面四层结构，不能省略，也不要写成长文。

# 检索结果
- 时间范围：YYYY-MM-DD 00:00 至 YYYY-MM-DD 23:59
- 本次找到：X 场符合条件的活动
- 城市分布：深圳 X｜广州 X｜珠海 X｜香港 X｜澳门 X
- 最值得优先看：活动名1；活动名2；活动名3（不足3个就如实写）

# 候补链接
这个区域专门放“接近但未正式入选”的活动，尤其是深圳本地活动。不要写长解释，每条只保留超短格式：
- 活动名｜城市｜未入选原因关键词｜链接

如果某次深圳正式入选为 0，必须优先给一个小节：
## 深圳候补（可自行判断）
然后列出 2-5 条深圳接近候选，只写短链接格式，不展开。

候补活动允许是这些情况：
1. 主题接近但不够纯
2. 主办方偏产业/偏产品但仍有一定技术含量
3. 高校活动真实但和计算机主题有一点距离
4. 时间地点明确，但技术硬度略低于主清单

# 备注
- 已剔除的典型原因：商业宣讲 / 时间地点不明 / 主题不够硬核 / 不在时间窗内 / 非公开活动（按实际写）
- 交通时间与费用：如无官方精确数据，可基于公开交通信息估算，并明确标注“估算”

八、字段填写规则
1. 日期统一 YYYY-MM-DD HH:MM。
2. 如果总时长无法直接得出，写“官方未明确”。
3. 如果报名方式不明确，写“原文未说明”。
4. 如果校外能否旁听不明确，写“官方未明确，建议联系主办方确认”。
5. “我能学到什么”必须具体，不能写空话，优先写开发者/科研/申请者真正能带走的内容。
6. “理由”必须简短具体，1-2句即可，不要写成长段评论。
7. 快速卡片区和完整档案区的活动顺序必须一致。
8. 候补链接区不要写大段文字，不要喧宾夺主。

九、交通与可达性
1. 固定起点：深大地铁站A3/A4出口。
2. 深圳本地：优先给地铁或地铁+步行方案。
3. 广州/珠海：仅保留单程大致可控制在2.5小时内、费用大致不高的高含金量活动。
4. 香港：仅保留单程大致1小时内可达、费用合理的高含金量活动；若略超但活动非常硬核，可保留并注明“时间为估算”。
5. 澳门：仅保留全程大致3小时内可达、费用合理的高含金量活动。
6. 交通不是一票否决项：若活动特别硬核且真实，但交通细节无法完全精确核验，可保留并注明“交通为公开交通信息估算，建议出发前复核”。

十、风格要求
1. 真实性优先于完整性。
2. 输出整洁、稳定、像一个长期可复用的活动情报模板。
3. 不要使用夸张措辞，不要写营销口吻。
4. 不要输出你不确信的内容；不确定就明确标注“官方未公开”或“未核实”。
